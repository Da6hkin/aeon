### Idea 1 — Regression Sentinel (executable)

**Thesis** (1 sentence): AI-authored PRs cause a hidden class of "fix-one-break-ten" regressions because reviewers only diff the target file — ship a GitHub check that replays golden-path traffic against the pre-vs-post branch and blocks any PR that silently changes a flow the author didn't touch.

**ICP** (role + trigger event): Head of Engineering at a 5–30-engineer B2B SaaS whose team just merged an AI-authored PR that caused a Slack #incidents thread within the last 14 days. Trigger event: the #incidents post-mortem lands in their inbox.

**Wedge** (first 12 months): A one-repo-scope GitHub App that (a) records real staging traffic (or a small synthetic seed set) into a "golden flows" file, (b) auto-runs those flows against the pre-vs-post AI PR sandbox, (c) posts a check "🟢 20 flows unchanged / 🔴 3 flows changed: /billing/cancel, /webhook/stripe, /api/v2/search". No hand-written tests, no test-authoring workflow — the golden set is inferred from real traffic. That's the single sharp product for year one.

**Pain evidence** (2+ permalinks):
  - *"Fix-one-break-ten pattern is the most frequently reported frustration, especially in codebases beyond a few hundred lines."* — https://www.morphllm.com/reddit-vibe-coding
  - *"After over half an hour of thinking, it produced a very ad-hoc implementation with needless limitations instead of properly refactoring."* (matheusmoreira, HN 47660925) — https://news.ycombinator.com/item?id=47660925
  - *"16 out of 18 CTOs reported production disasters directly caused by AI-generated code."* (Final Round AI Aug 2025 survey, cited via morphllm synthesis) — https://www.morphllm.com/reddit-vibe-coding

**Monetization**: $99/mo team (up to 500 golden flows, 1 repo), $499/mo scale (10 repos, 5,000 flows). Gross margin ~85% (customer's own staging bears compute). LTV target 24mo × $99 = ~$2.4k; CAC target ≤$400 → 6× LTV:CAC.

**Distribution** (specific channel + CAC estimate): Cold X DMs to 50 named devrel + founder handles at AI-native coding tools (Bolt, Lovable, Cursor, Windsurf, Codex, Replit Agents) offering white-label integration — one integration pipes 10–100× the raw pilots. Secondary: Show HN when the golden-flow inference hits >90% coverage on a real OSS repo. CAC est. $250–400 per paid seat via named cold-outbound.

**Moat** (what compounds): Golden-set inference from a customer's own traffic — the longer they run it, the sharper the regression boundary. Each customer's corpus is proprietary to them and to Regression Sentinel; leaving costs them the calibrated boundary. Workflow lock-in (PR check becomes required-for-merge on Day 30).

**Why now (2026)**: In H1 2026 the majority of PRs in early-stage teams became AI-authored (Bolt/Lovable/Cursor/Claude Code/Codex), and the fix-one-break-ten cost is now visible weekly in Slack #incidents channels. GPT-5.6 launched 2026-07-11 with aggressive multi-file edits — the regression surface just got bigger. Cost-curve shift: AI generation is now cheaper than review; the reviewer bottleneck is a business.

**MVP test** (2 weeks): Ship as a GitHub App on 3 pilot repos volunteered from the DM cohort. Metric: on ≥5 AI PRs per pilot, does the check flag a regression the human reviewer would have missed? Success = ≥1 saved-incident-per-pilot in 30 days.

**Kill criteria** (numeric): <3 paid pilots ($99/mo each) by Day 60, OR flagged-regression precision <60% (false-alarm rate too high for customers to keep the check as required-for-merge).

**Expansion** (if it works): (a) IDE plugin that shows the regression check inline in Cursor/Windsurf/Claude Code before the PR is opened; (b) LLM-agent evaluator that auto-writes the fix for a flagged regression before human review; (c) sell into the AI coding tools themselves as an OEM safety layer (Bolt, Lovable — a "verified merge" checkbox).

### Idea 2 — Attestly (ambitious)

**Thesis** (1 sentence): Every agent-to-agent commerce protocol (Virtuals ACP, x402r, AP2) needs neutral third-party evaluators to release escrow — Attestly becomes the marketplace of specialized evaluator agents that every ACP transaction plugs into, taking a fee on every attestation and compounding a proprietary "was-the-deliverable-good" dataset.

**ICP** (role + trigger event): Buyer agents (and their human operators) on Virtuals ACP transacting for code, translation, or creative-asset deliverables. Named initial cohort: the ~200 Virtuals subnet operators live on ACP public beta as of 2026-07-03, plus the Base agent marketplaces that launched in Q2. Trigger event: they attempt a first ACP transaction and hit the "who evaluates?" question.

**Wedge** (first 12 months): Three vertical evaluator agents shipped as pluggable Virtuals ACP evaluators: (a) **code delivery** — runs the buyer's specified test suite in a sandboxed runner + static analysis + diff-vs-spec check; (b) **translation delivery** — round-trip embedding + BLEU + spot-check LLM judge on 5 sampled sentences; (c) **creative-asset delivery** — CLIP-score vs written spec + rubric-based LLM judge. Each ships with a documented ≥90% agreement rate vs human evaluators on 100 seed transactions before going live.

**Pain evidence** (2+ permalinks):
  - *"evaluator agent needs to determine whether the agent's deliverable truly satisfies the original user request — a non-trivial problem when both requests and outputs are expressed in natural language. Evaluator agents might struggle to confidently validate whether a deliverable meets expectations."* — https://cloud.google.com/customers/virtualsprotocol
  - *"When Agent B finishes the work, a third-party oracle or evaluator agent checks that the results match the agreed criteria."* (Delphi Digital, Oct 16 2025) — https://members.delphidigital.io/reports/virtuals-acp-open-coordination-for-digital-labor
  - *"the Registry, as currently deployed, cannot function as a trust signal: values are not commensurable, feedback records are rarely grounded in verifiable interactions."* (Xiong et al. arXiv 2606.26028, rev 2026-07-08) — https://arxiv.org/pdf/2606.26028

**Distribution** (specific channel + CAC estimate): (1) Partner directly with Virtuals Protocol to become the default evaluator for the 3 seed verticals — their own docs admit the gap. (3) Regulatory positioning as the "neutral third party" for agent transactions has long-run defensibility.

**Kill criteria** (numeric): <3 paying marketplace integrations at $99/mo each by Day 90, OR evaluator-human agreement rate <85% averaged across the 3 verticals.

**Expansion** (if it works): (a) Reputation aggregator built on top of accumulated attestations — becomes the trust layer of the agent economy, filling the "reputation aggregator API" gap that ERC-8004 leaves open; (b) evaluator-agents-for-hire outside ACP (any AI system that needs "did the agent do what I asked" verification — RLHF pipelines, LLM app QA, agent CI/CD); (c) partner with Coinbase-for-Agents / AgentCore Payments as the reference evaluator for x402r settlement.
